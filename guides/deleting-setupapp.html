Deleting Setup.app

Overview

Deleting Setup.app is an universal method to bypass activation on 32-bit iOS devices in case a specific device and iOS version combination is not supported by any other tool.

Warning: It is strongly recommended to use any other method for bypassing activation, as deleting setup.app will break:
  • iTunes syncing
  • Siri
  • Notifications
  • and possibly iCloud sign-in.

Prerequisites

What You'll Need

A computer running Linux or macOS with the following installed:

  • An SSH ramdisk creation and booting tool, like Legacy-iOS-Kit or SSHRD_Script
  • An SFTP client like FileZilla

Step-by-Step Guide

Step 1: Ramdisk creation

Select one of the tools below to show the matching ramdisk creation instructions.

Legacy-iOS-Kit Instructions

  1. Download Legacy iOS Kit and extract the zip archive.
  2. Open a Terminal window.
  3. If on macOS, install Xcode Command Line Tools by running this command: xcode-select --install
  4. Connect your device to your computer.
  5. Go to where the extracted files are located, drag restore.sh to the Terminal window, and press Enter/Return.
  6. Go to Useful Utilities -> SSH Ramdisk
  7. Follow on-screen instructions to create and boot an SSH ramdisk.
    8. Info: You can find your iOS version by using iDescriptor (all platforms) or Legacy-iOS-Kit (Linux/macOS)
    Warning: A5 devices will require the use of checkm8-a5 to enter pwnDFU mode.
  8. Once the device has booted the SSH ramdisk, select Connect to SSH in the terminal window.
  9. Finally, run mount.sh to mount the filesystems.

SSHRD_Script Instructions

  1. Clone the SSHRD_Script repository and enter it using the terminal: git clone https://github.com/verygenericname/SSHRD_Script --recursive && cd SSHRD_Script.
  2. Run ./sshrd.sh [iOS version for ramdisk].
    3. Info: You can find your iOS version by using iDescriptor (all platforms) or Legacy-iOS-Kit (Linux/macOS)
  3. Run ./sshrd.sh boot to boot the ramdisk.
  4. Run ./sshrd.sh ssh to connect to SSH on your device.
  5. Mount the filesystems by running mount_filesystems.

Step 2: Remove setup.app

In the SSH terminal, run this command to rename setup.app:

  • mv /mnt1/Applications/Setup.app /mnt1/Applications/Setup.app.bak

Step 3: Reboot

Reboot your device to apply the changes by either holding the power and home buttons for 10 seconds or running /sbin/reboot. Your device should now boot into the home screen.
